#  Deep Learning-Based Multi-User Detection in Underwater Optical NOMA Networks

A MATLAB-based simulation of a **1D Convolutional Neural Network (CNN)** receiver for multi-user detection in Underwater Optical Wireless Communication (UWOC) systems using Non-Orthogonal Multiple Access (NOMA). The CNN-based decoder is benchmarked against a traditional Successive Interference Cancellation (SIC) receiver across multiple water types and transmission distances.

## Overview

Underwater optical communication offers high-speed, low-latency data transfer for applications such as:

-  Autonomous Underwater Vehicles (AUVs)
-  Ocean monitoring and environmental sensing
-  Offshore exploration
-  Underwater robotics

**The problem:** UWOC channels suffer from severe attenuation, scattering, and turbulence. When combined with NOMA (where multiple users share the same optical channel via power-domain multiplexing), traditional SIC receivers degrade significantly in turbid water.

**The solution:** A 1D-CNN trained on simulated channel data learns to directly map received waveforms to decoded bits — without requiring explicit channel estimation.

```
X_tx = √P₁·x₁ + √P₂·x₂
```

| User   | Role      | Power Allocation |
|--------|-----------|-----------------|
| User 1 | Far user  | P₁ = 0.8        |
| User 2 | Near user | P₂ = 0.2        |

## Channel Model

### Beer–Lambert Attenuation

```
h_l = exp(−c · d)
```

Where `c` = attenuation coefficient, `d` = transmission distance.

| Water Type     | Attenuation Coefficient `c` (m⁻¹) |
|----------------|-----------------------------------|
| Pure Sea Water | 0.056                             |
| Clear Ocean    | 0.150                             |
| Coastal Water  | 0.398                             |

### Turbulence + Noise

Turbulence is modeled as a lognormal random variable `T`. The received signal is:

```
Y = h_l · T · X_tx + n
```

Where `n` is AWGN with zero mean.

## CNN Architecture

```
Input Layer        →  Received signal samples
Conv1D (16 filters) →  ReLU activation
Conv1D (32 filters) →  ReLU activation
Fully Connected    →  Dense layer
Output Layer       →  Sigmoid → Decoded bits (User 1 & User 2)
```

- **Optimizer:** Adam
- **Loss:** Binary Cross-Entropy
- **Epochs:** 20
- **Batch Size:** 128

## Results

### BER vs. Distance
- The **SIC receiver** begins to fail beyond ~15 m in high-turbidity (coastal) water.
- The **CNN receiver** maintains significantly lower BER up to 25 m across all water types.

### Eye Diagram
- **Pure sea water:** Eye diagram remains wide open — good signal integrity.
- **Coastal water:** Significant eye closure — SIC threshold detection fails; CNN adapts.

### CNN Training
- Training loss decreases and validation accuracy increases steadily over 20 epochs.
- No significant overfitting observed with the 70/15/15 split.

>  Figures (BER curves, eye diagrams, training plots) are generated by the MATLAB simulation scripts.

## Getting Started

### Prerequisites

- MATLAB R2021a or later
- MATLAB Deep Learning Toolbox
- Signal Processing Toolbox

### Running the Simulation

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/uwoc-noma-cnn.git
   cd uwoc-noma-cnn
   ```

2. **Generate the dataset**
   ```matlab
   run('scripts/generate_dataset.m')
   ```

3. **Train the CNN**
   ```matlab
   run('scripts/train_cnn.m')
   ```

4. **Run BER comparison**
   ```matlab
   run('scripts/ber_comparison.m')
   ```

5. **Plot results**
   ```matlab
   run('scripts/plot_results.m')
   ```
